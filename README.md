# 📱 KoreanSongLesson — мобильная версия (PWA)

Автономное веб-приложение: из текста корейской песни делает учебный разбор на русском
(2 грамматики + слова/фразы + озвучка) **прямо в браузере телефона, офлайн**. Корейский
морфоразбор выполняет **Kiwi** (WebAssembly, модель ~35 МБ) — без сервера и без интернета
после первой загрузки. Логика разбора — тот же движок, что в десктопной версии
(порт на JS, выверен число-в-число против Python).

## Что внутри
- `index.html`, `js/` — приложение (UI + порт анализатора/урока/рендера).
- `lib/` — kiwi-nlp 0.20.4 (WASM + обвязка).
- `model/` — языковая модель Kiwi (knlm, ~54 МБ; можно уменьшить, см. ниже).
- `data/` — базы: 46 грамматик, ~2400 слов, 72 фразы.
- `manifest.webmanifest`, `sw.js`, `icons/` — PWA (установка на экран + офлайн-кэш).
- `serve.py` — локальный сервер для запуска/раздачи.

## Как проверить

### На компьютере (проще всего)
```powershell
cd X:\_CLOUDE\KoreanSongLesson\mobile
python serve.py            # http://127.0.0.1:8090
```
Открой **http://127.0.0.1:8090** в Chrome. В адресной строке появится значок «Установить» —
можно поставить как приложение. На `localhost` работает офлайн-кэш и установка.

### На телефоне в той же WiFi
```powershell
python serve.py 8090 all   # печатает адрес http://<IP-компьютера>:8090
```
На телефоне открой показанный `http://<IP>:8090`. Приложение **работает** (разбор, озвучка).
⚠ Но «установка на экран» и офлайн-кэш браузер разрешает только на **HTTPS или localhost** —
по обычному `http://<IP>` этого не будет (это требование браузеров к PWA, не баг).

### Настоящее приложение на телефоне (установка + офлайн)
Нужен HTTPS. Всё считается на стороне клиента, сервер не требуется, поэтому достаточно
выложить статику на любой **бесплатный HTTPS-хостинг**.

**Самый быстрый способ без регистрации — Netlify Drop:**
1. Открой на компьютере **https://app.netlify.com/drop**
2. Перетащи туда папку `mobile/` (или архив `KoreanSongLesson-mobile.zip` из корня проекта).
3. Получишь ссылку вида `https://…netlify.app` — открой её **на iPhone в Safari**.
4. В Safari: **Поделиться → «На экран Домой»**. Готово — работает как приложение, офлайн
   (первая загрузка ~39 МБ модели — один раз, дальше из кэша).

Альтернатива с постоянным адресом — **GitHub Pages** (нужен аккаунт GitHub): создать репозиторий,
залить туда содержимое `mobile/`, включить Pages (Settings ▸ Pages ▸ ветка). Модель `sj.knlm`
(35 МБ) < лимита GitHub 100 МБ/файл — проходит.

Модель уже «тонкая» (~39 МБ, без словарей имён собственных) — это важно для памяти и
офлайн-кэша на iPhone. Совместимость и работа модели проверены в Node (kiwi-nlp 0.20.4).

## Возможности
- **🔊 Озвучка** корейского (голос телефона, офлайн).
- **Скачать HTML** урока и **Anki (.txt)** — импортируется в Anki (Файл ▸ Импорт, разделитель Tab).
- **Примеры** песен и **поиск текста по названию** (LRCLIB; на части хостингов может блокироваться CORS — тогда вставляй текст вручную).
- Всё офлайн, ничего не отправляется на сервер.

## Уменьшить размер модели (опционально)
Сейчас грузятся все 8 файлов модели. Можно убрать `default.dict` (3 МБ) и `multi.dict` (12 МБ)
— это словари имён собственных, для песен не критичны. В `js/app.js` в `MODEL_FILES` убери эти
два имени и в `build({...})` добавь `loadDefaultDict:false, loadMultiDict:false`. Останется ~39 МБ.

## Техническое
- kiwi-nlp 0.20.4 (WASM, одно-поточный — спец-заголовки COOP/COEP не нужны).
- Морфемные якоря (MORPH_ANCHORS) + валидация по морфемам + регэкспы — идентичны Python-версии.
- Токены Kiwi: `{str, tag, position, length}` → нормализуются в `{form, tag, start, end, lemma}`.
